# Miao 桌面窗 UI 规格

本规格以用户最新确认的猫头额度面板为唯一实现基准。产品不包含独立全身宠物或用户可切换的展开态，只保留猫头异形窗、最小尺寸响应式排版与局部表情动画。

## 1. 首屏允许出现的文案

所有首屏固定文案必须同时提供简体中文和英语，首次启动跟随系统语言，并允许用户在天气设置弹层中切换。下面表格列出中文版语义基准；英文不得改变信息层级或挤出猫形安全区。

| 区域 | 固定文案 | 动态内容 |
| --- | --- | --- |
| 标题 | `Miao` | 账号会员状态，如 `PLUS 会员` |
| 额度列表 | `已用`、`剩余`、`重置` | 按 `window_minutes` 生成周期名称、百分比、重置时间、倒计时 |
| 页脚 | `设置天气` | 城市、气温、天气状态、体感温度、湿度、风速 |
| 无数据 | `演示账号` | 演示额度；必须明确标注，禁止伪装成真实数据 |

除错误提示、可访问性文本和系统托盘菜单外，不在首屏新增营销文案、说明卡片或装饰性徽章。

## 2. 布局与容器

1. 设计坐标为 `520 × 460`，默认以 50% 的 `260 × 230` 启动；窗口透明、无系统边框、默认置顶，并允许在 25%–150% 范围内等比缩放。
2. 唯一主容器是猫头外壳：顶部两只耳朵，中部圆润脸颊，下缘轻微收窄。
3. 耳朵与标题区属于可拖动区域；右下角缩放手柄和天气入口必须排除窗口拖动。
4. 标题下方只显示猫眼、鼻口与胡须，不出现独立全身猫或额外宠物窗口。
5. 额度区使用一个内嵌面板；一个周期时自动居中，多个周期时用分隔线区分，不拆成多张卡片。
6. 36% 以上按完整面板等比缩放；25%–36% 自动采用可读性更高的内部排版，猫形轮廓始终等比且不得拉伸。

## 3. 设计令牌

| 令牌 | 值 | 用途 |
| --- | --- | --- |
| `ink-950` | `#031a18` | 猫头主背景 |
| `ink-900` | `#072724` | 内嵌额度面板 |
| `ink-800` | `#123936` | 控件与分隔线 |
| `mint-400` | `#61e8ad` | 外轮廓、在线状态、剩余百分比 |
| `mint-300` | `#8bf4c7` | 高亮与悬停 |
| `coral-400` | `#ff6475` | 剩余低于 30% 的额度警示与错误状态 |
| `warning-400` | `#f2c14e` | 剩余 30%–59% 的额度警示 |
| `track-500` | `#405674` | 进度槽 |
| `text-100` | `#f7faf8` | 主文字 |
| `text-400` | `#9fb4c0` | 重置时间与页脚 |
| 外壳描边 | `2px solid rgba(97,232,173,.9)` | 猫头轮廓 |
| 外壳柔光 | 描边局部 `drop-shadow` | 与桌面分离且不生成矩形透明层 |

背景锁定为冷调深墨绿色，不改成纯黑、奶油白或暖灰。额度状态色只使用薄荷绿、警示黄与珊瑚红。

## 4. 字体与图标

- 字体栈：`Inter, "Microsoft YaHei UI", "PingFang SC", system-ui, sans-serif`。
- 标题 22px/700；标题副文案 17px/650；额度标题 19px/700；百分比 18px/700；重置时间 15px/500；页脚 13px/500。
- 数字启用等宽数字特性，刷新时避免布局跳动。
- 图标仅保留时钟、天气与状态圆点；宠物窗口不显示刷新或关闭图标。

## 5. 动效与状态

| 状态 | 局部动画 | 界面反馈 |
| --- | --- | --- |
| 正常闲置 | 约每 3.8–9 秒随机眨眼，24% 概率连续眨两次；耳朵约每 6.5–14 秒轻摇一次 | 状态点常亮 |
| 正在读取 | 眼睛保持自然状态，耳朵与面板不跳动 | 新额度到达后进度条柔和过渡 |
| 无本地数据 | 保留眨眼与摇耳 | 显示明确的演示模式标记 |
| 读取失败 | 保留眨眼与摇耳 | 保留最后一次合法额度快照 |
| 天气变化 | 眨眼与摇耳不受影响；天气层低频漂移 | 晴/云/雾/雨/雪/雷雨映射到裁剪在猫身内的背景层 |

所有循环动效遵守 `prefers-reduced-motion`；降低动效时停止随机调度和天气背景位移。

## 6. 核心交互

1. 启动后读取最新 Codex `rollout-*.jsonl` 中的 `payload.rate_limits`。
2. 监听本地会话目录变化并防抖刷新；同时保留 60 秒兜底轮询。
3. 宠物窗口不提供手动刷新按钮，额度值随 Codex 写入的结构化事件自动同步。
4. 系统托盘以“显示 Miao”复选项控制显示或隐藏，并提供“退出 Miao”。
5. 拖动右下角缩放手柄等比改变窗口尺寸，最小 25%、默认 50%、最大 150%；键盘方向键也可逐级调整。
6. 数据读取只发生在 Electron 主进程；渲染进程通过窄 IPC 获取结构化结果，不接触用户文件系统。
7. 不读取 `auth.json`，不上传会话内容，不解析提示词正文，只提取额度对象、模型名和事件时间。
8. 点击底部天气区域设置城市；未设置时不联网，设置后由主进程向 Open-Meteo 查询气温、体感温度、湿度、风速和天气状态，并每 30 分钟刷新。

## 7. 验收基准

- 猫耳、猫脸轮廓、动态额度层级、天气与会员状态和用户确认图一致。
- 页面中没有独立全身猫、紧凑态摘要、收起按钮或第二种窗口形态。
- 眨眼节奏自然，不做机械连续闪烁；两只耳朵低频、短促且不同向轻摇。
- 真实模式与演示模式不可混淆。
- 额度条、百分比和重置时间在 0%、100%、跨日和已过期场景下均不溢出。
- 剩余 60%–100% 为绿色、30%–59% 为黄色、0%–29% 为红色；边界值由单元测试锁定。
- 5 小时额度缺失时不显示占位或演示值，其他真实周期继续正常展示。
- 天气动态层不得越出猫形轮廓，不得降低额度文字和进度条可读性。
- 窗口透明区域无白底；主要内容无裁剪、意外换行或浏览器默认控件字体。
- 25%、50%、100% 与 150% 四档尺寸下，标题、会员状态和天气详情均保持在猫形安全区内。
- 设计稿与最终截图必须使用 `view_image` 同轮对照，并记录至少五项视觉差异处理。
